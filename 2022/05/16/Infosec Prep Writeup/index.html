靶机比较简单。

1. 靶机信息

靶机地址：

  • InfoSec Prep: OSCP ~ VulnHub

目标：

  • 拿到root权限
  • 拿到1个flag

难度：

  • 简单

2. 信息收集

2.1. 端口信息

1. sudo arp-scan -l
2. sudo nmap -p- 10.0.2.49
3. sudo nmap -p22,80,33060 -A 10.0.2.49

33060端口上的服务未知。

80端的服务为http，并且存在robots.txt

2.2. 网页信息

可以看到网页的CMS是wordpress。并且还给了一个提示 

Oh yea! Almost forgot the only user on this box is “oscp”.

也就是该靶机上可能存在一个用户 oscp。

3. get shell

从信息收集中得到了靶机上可能存在一个用户 oscp，并且一个密钥，尝试使用密钥ssh登录。 

ssh oscp@10.0.2.49 -i id_rsa

登录成功

4. 权限提升

4.1. 方法1

通过信息收集知道，目标靶机的操作系统为Ubuntu 20.04，因此直接使用CVE-2021-3493，一步到位

但是，该靶机的发布时间是2020年，该CVE还没有被披露。所以，肯定存在另一种办法提权。

4.2. 方法2

利用下面的命令查找具有suid权限的文件： 

find / -perm -u=s -type f -ls 2>/dev/null

suid bash提权命令： 

bash -p

最后读取器flag 

// /root/flag.txt
d73b04b0e696b0945283defa3eee4538

5. 总结

靶机难度简单，涉及以下的攻击方法：

  1. ssh密钥登录
  2. 内核漏洞
  3. suid提权
