suid, ruid, euid 区别

发表于 2022-05-17 分类于 Linux 阅读次数： Disqus： 本文字数： 2k 阅读时长 ≈ 2 分钟

个人知识点记录。

1. 解释

  • RUID(Real User ID)
    • The actual owner of a process
    • It is used in signal transmission. A unprivileged process can signal to the another process when the RUID, EUID is the same as RUID, SUID of the another process
  • EUID(Effective User ID)
    • Generally, UID and EUID is the same
    • EUID is changed by executable file that is configured SetUID authority
    • EUID temporarily stores another account's UID
    • The authority of a process is determined according to the UID stored in the EUID
  • SUID(Saved set-user-ID)
    • SUID is used when a process'authority is recovered after lowered
    • When process'authority is changed to lower, previous EUID is stored at EUID
    • Then, when the lowered authority is rcovered to original authority, the SUID is stored at EUID

2. 理解

Linux系统中并不是通过用户名来判断用户的权限的，而是通过uid。在提权过程中经常会遇到过这么一种情况，那就是普通用户对/etc/passwd文件有写权限，我们就可以利用普通用户的写权限往/etc/passwd中添加类似下面的内容：

其中，第一个0表示uid，第二个0表示gid。这就代表着我们创建了一个用户名为m1ku的用户，并且uid和gid都为0。那么即使该用户名不是root，它也拥有root权限。因为Linux中uid=0表示的就是root。

  • RUID, 用于在系统中标识一个用户是谁，当用户使用用户名和密码成功登录后一个UNIX系统后就唯一确定了他的RUID.
  • EUID, 用于系统决定用户对系统资源的访问权限，通常情况下等于RUID。
  • SUID，用于对外权限的开放。跟RUID及EUID是用一个用户绑定不同，它是跟文件而不是跟用户绑定。

但是普通用户可以通过passwd修改密码，也就是修改/etc/shadow中的内容。那么这到底是如何实现的呢？为什么普通用户能够修改一个没有权限的文件的内容呢？这是因为 passwd 这个文件的权限。

从上图中可以看到，原来属主的'x'权限所在的地方变成了's'，这个's'表示的就是SUID。接下来分析一下普通用户执行 passwd 命令的过程：

  1. 普通用户hdfzzf在shell中执行 passwd
  2. 系统判断hdfzzf是否有执行的权限，如果有则下一步，没有则 permission denied；
  3. 系统判断 /bin/passwd 是否具有suid，如果有则下一步，没有则以hdfzzf权限执行，也就是euid为hdfzzf，跳到5；
  4. 执行passwd，并且权限为root（属主），此时该进程的euid为root；
  5. 输入新的密码，如果passwd进程的euid为root，则将密码写入/etc/shadow，否则无法写入，密码修改失败；
  6. 结束进程
