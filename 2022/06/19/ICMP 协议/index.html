ICMP协议

hdfzzf
2022-06-19
计算机基础 › 计算机网络
协议, 计算机网络

个人知识点记录。

1. 简介

Internet Control Message Protocol (ICMP)，Internet 控制报文协议，它工作在网络层，用于在主机、路由器之间传递控制信息。控制信息有网络通不通、主机是否可达、路由是否可用等信息。这些控制信息中不会携带用户的数据，但对用户的数据传输有着重要作用。

虽然 ICMP 工作在网络层，但它是基于 IP 数据报的。

在 IP 通信中，会发生如下几种情况：

  • 中间某个路由器的缓冲区溢出，将分组丢弃
  • 到达了目标主机，但 checksum 或者端口出错，分组被目标主机丢弃

在这种情况下，发送方需要知道分组为什么被丢弃，在哪里被丢弃等信息，重传的时候可以作为参考。一只“信鸽”将这些信息从“丢弃现场”带回到发送方，这只“信鸽”就是 ICMP。

2. 作用

ICMP 大致分为两种功能：差错通知 和 信息查询。

  1. 给发送方的差错通知：某个中间路由器或者目标主机（称为A）接收到该分组，但是在处理过程中发生了错误（缓冲区溢出、checksum 错误等），导致分组被丢弃时，A需要发送 ICMP 报文告知发送方错误的发生以及发生错误的原因。
  2. 发送方的信息查询：发送方向目标主机发送 ICMP 请求报文，目标主机响应 ICMP 响应报文。被询问内容的种类非常丰富，最基本也是最常见的就是“查询目标主机是否可达”，比如我们平常使用的 ping。

3. 报文格式

ICMP 报文的内容放在 IP 数据包的数据部分。只从报文的格式来说明，ICMP 是 IP 的上层协议。但实际上 ICMP 和 IP 同属于网络层，ICMP 只是分担了 IP 的部分工作。可以这么理解，ICMP 报文是 IP 数据报的额外的头部。

将 ICMP 报文单独取出：

  1. Type: 定义了 ICMP 报文的类型
  2. Code: 发送这个 ICMP 报文的原因
  3. checksum: 差错检验
  4. 首部的其余部分对于不同的 ICMP 是不同的
  5. 数据部分对于差错报告报文是用于找出引起差错的原始分组的信息, 对于查询报文是基于查询类型的额外的信息

Type：

  • 0: Echo Reply
  • 3: Destination Unreachable
  • 4: Source Quench
  • 5: Redirect Message
  • 8: Echo Request
  • 9: Router Advertisement
  • 10: Router Solicitation
  • 11: Time Exceeded
  • 12: Parameter Problem: Bad IP header
  • 13: Timestap
  • 14: Timestap Reply
  • 15: Information Request
  • 16: Information Reply
  • 17: Address Mask Request
  • 18: Address Mask Reply

其中最常见的有：

  • ping操作中的请求和应答 (Type=8, Code=0 and Type=0, Code=0)
  • 目标不可达报文 (Type=3)
  • 超时报文 (Type=11, Code=0)
  • 时间戳请求/应答 (Type=13, Code=0 and Type=14, Code=0)
