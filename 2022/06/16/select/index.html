1. 简介

主要从socket网络编程方面来说明。最常用的一个函数就是：

参数：rlist, wlist, xlist 都是由“waitable objects”组成的可迭代的序列。可以是代表文件描述符的整数，或者是类似的 file-like 的对象，比如说 socket。

  • rlist：监听 rlist 中的对象，直到可以读取监听对象的内容
  • wlist：监听 wlist 中的对象，直到可以往监听对象中写入数据
  • xlist：监听 xlist 中的对象，直到监听对象发生了异常

返回值：

  • readable：如果 rlist 中的对象可以读取，则会出现在 readable 中
  • writable：如果 wlist 中的对象可以写入，则会出现在 writable 中
  • exceptional：如果 xlist 中的对象出现异常，则会出现在 exceptional 中

假设以socket为例，我作为server监听了与多个client的socket，这些socket都放在 rlist 之中，那么，只要 client 给我发消息，那么缓冲区就会有消息。那么在 readable 中就会出现给我发消息的 client 的 socket。

同样的，把需要响应的socket加入到 wlist 中，就可以从 writable 中看到对应的 socket，因为默认socket就是可写的。xlist 一般设置为和 rlist 一样，这样只要 rlist 中的socket出异常，就可以在 exceptional 中看到。也可以不设置。

2. 例子

下面server中实现的功能：接收来自 client 的数据，并原封不动的回复。

2.1. server

  1. server 先是创建一个socket（假设称之为 server），然后等待client的请求；
  2. 利用 readable, writable, exceptional = select.select(inputs, outputs, inputs) 监听 inputs 中是否有可读取的对象，outputs中是否有可写入的对象以及 inputs 中是否有出现异常的对象。
  3. 服务器从socket收到信息只有两种：请求连接或者数据通信。for s in readable 中：
    1. 如果 s 是 server，说明 server 有可以读取的内容，那必定就是有 client 向 server 发送了连接请求。因此，需要接收该请求，并将对方的 socket（称为 client）加入到 inputs 中进行监听，并为 client 创建一个消息队列
    2. 如果 s 不是 server，那么就说明，这是一个 client 向服务器发送数据，那么 server 需要做的就是判断收到的消息是否为空，
      • 如果不为空，说明 client 与 server 进行正常通信，server 先将收到的消息加入到 client 的消息队列，然后判断这是不是该 client 第一次与 server 通信，如果是则把该 client socket 加入到 outputs。
      • 如果消息为空，那么说明 client 想要与 server 断开连接，那么 server需要从 outputs（如果 client 存在），inputs中将 client 去掉，并且关闭连接
  4. for s in writable 用来处理是否有消息需要处理：
    1. 首先取出某个 client 的 socket，并且从消息队列中取出需要发送给 client 的消息（可以用队列实现，不过我实现的时候出了一点问题）然后发送，如果取完值之后消息队列为空，那么将该消息队列删除。
    2. 如果取值之前队列就为空，那么就出现异常
  5. for s in excetpitonal 用来处理 inputs 中的对象，也就是和client的socket出现的异常

2.2. client

  1. 首先定义需要发送的消息，并且创建socket对象
  2. 第一个for循环中，与server建立了两个TCP连接
  3. 第二个for循环(for index, message in enumerate(messages)) 中，向 server 发送定义好的消息
  4. 第三个for循环(for s in socks)中，依次从不同的socket中接收消息，并且在接受完全部的消息之后（一次性全部接收）关闭连接

这里有一个bug，那就是有可能server回复慢，导致有一两条消息还没接收就关闭了连接，这里可以改进。不过这里演示的是 select 模块的使用，所以就这样吧。
