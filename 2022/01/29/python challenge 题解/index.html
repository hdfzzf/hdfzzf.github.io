level 1

题目信息

题目信息如下：

根据图片应该是某种加密方法，图中的 K->M, O->Q, E->G，正好符合规律：K往后两个字母就是M，O往后两个字母就是Q，E往后两个字母就是G。猜测只要把下面粉色的字母全部用自己的后两个字母代替即可。

方法 1

代码： 

encypt_str = "g fmnc wms bgblr rpylqjyrc gr zw fylb. rfyrq ufyr amknsrcpq ypc dmp. bmgle gr gl zw fylb gq glcddgagclr ylb rfyr'q ufw rfgq rcvr gq qm jmle. sqgle qrpgle.kyicrpylq() gq pcamkkclbcb. lmu ynnjw ml rfc spj. " # 原来的字符串  
  
shift = 2 # 后移步数  
new_str = "" # 新的字符串  
  
for ch in encypt_str:  
    if str.isalpha(ch): # 判断是否是字母  
        ch = ord(ch)
        ch = ((ch + shift) - 97) % 26 + 97 #  y 和 z 变成 a 和 b
        ch = chr(ch)  
    new_str += ch  
print(new_str)
（需要注意的是 y 和 z 变成 a 和 b 的时候！！！）

结果： 

i hope you didnt translate it by hand. thats what computers are for. doing it in by hand is inefficient and that's why this text is so long. using string.maketrans() is recommended. now apply on the url.

发现除了提示我们可以使用 maketrans() 方法之外，没有其他信息。我们回到 url 

http://www.pythonchallenge.com/pc/def/map.html

方法 2

maketrans() 方法介绍

Python maketrans() 方法用于创建字符映射的转换表，对于接受两个参数的最简单的调用方式，第一个参数是字符串，表示需要转换的字符，第二个参数也是字符串表示转换的目标。 注：两个字符串的长度必须相同，为一一对应的关系。python 3 和 python 2 的使用方法不同，2 需要导入 string，而 3 不需要。

  • 语法： str.maketrans(intab, outtab)

  • 参数：

    • intab：字符串中要替代的字符组成的字符串。
    • outab：与 intab 一一对应的映射字符的字符串。

  • 返回值： 返回字符串转换后生成的新字符串。

  • 一般配合 translate() 使用

translate() 方法介绍

Python translate() 方法根据参数table给出的表(包含 256 个字符)转换字符串的字符, 要过滤掉的字符放到 del 参数中。

  • 语法： str.translate(table[, deletechars]);

  • 参数：

    • table：翻译表，翻译表是通过maketrans方法转换而来。
    • deletechars：字符串中要过滤的字符列表。

  • 返回值： 返回翻译后的字符串。

  • 本题演示： 

    encypt_str = "g fmnc wms bgblr rpylqjyrc gr zw fylb. rfyrq ufyr amknsrcpq ypc dmp. bmgle gr gl zw fylb gq glcddgagclr ylb rfyr'q ufw rfgq rcvr gq qm jmle. sqgle qrpgle.kyicrpylq() gq pcamkkclbcb. lmu ynnjw ml rfc spj. "

intab = "abcdefghijklmnopqrstuvwxyz"  
outtab = "cdefghijklmnopqrstuvwxyzab"  
trantab = encypt_str.maketrans(intab, outtab)  
  
print(encypt_str.translate(trantab))

方法 3

zip() 方法介绍

zip() 函数用于将可迭代的对象作为参数，将对象中 对应的元素 打包成一个个元组，然后返回由这些元组组成的对象，这样做的好处是节约了不少的内存。 我们可以使用 list() 转换来输出列表。也可以用 dict() 转换为字典，此时就产生了映射关系。 如果各个迭代器的元素个数不一致，则返回列表长度与最短的对象相同，利用 * 号操作符，可以将元组解压为列表。

  • 语法： zip([iterable, ...])

  • 参数：

    • iterabl：一个或多个迭代器

  • 返回值： 返回一个对象。

  • 解压： zip(*被压缩的对象)，返回二维矩阵式

  • 本题演示： 

    encypt_str = "g fmnc wms bgblr rpylqjyrc gr zw fylb. rfyrq ufyr amknsrcpq ypc dmp. bmgle gr gl zw fylb gq glcddgagclr ylb rfyr'q ufw rfgq rcvr gq qm jmle. sqgle qrpgle.kyicrpylq() gq pcamkkclbcb. lmu ynnjw ml rfc spj. " # 原来的字符串

intab = "abcdefghijklmnopqrstuvwxyz,. '()"  
outtab = "cdefghijklmnopqrstuvwxyzab,. '()"  
transtab = dict(zip(intab, outtab))  
print("".join(transtab[x] for x in encypt_str))

level 3

题目信息

除了一句提示，什么都没有。查看页面源码，发现字符串：

根据提示做出合理猜测：找出这一字符串中的小写字母，每个小写字母的前后都是三个大写字母！！！也就是 AAAbCCC 类型的

方法

步骤：

  1. 请求网址，拿到页面源码
  2. 取出这一大串字符串
  3. 从中找出符合条件的小写字母

代码：

import requests, re  
url = 'http://www.pythonchallenge.com/pc/def/equality.html'  
r = requests.get(url)  
tar = re.findall(r'<!--(.*?)-->', r.text, flags = re.S)[-1] # 取出目标字符串  
ch_lst = re.findall(r'[^A-Z]+[A-Z]{3}([a-z])[A-Z]{3}[^A-Z]+', tar, re.S)    # 从目标字符串中取出符合条件的小写字母  
print("".join(ch for ch in ch_lst)) # 拼接成 url

解释： 主要说明一下取出符合条件的小写字母的模式是如何而来的。

[a-z] 表示一个小写字母；

[A-Z] 表示一个大写字母；

[A-Z]{3} 表示三个大写字母；

[A-Z]{3}[a-z][A-Z]{3} 表示 ...AAAbCCC... 类型的，注意 ... 可能为大写，比如说：AAAAbCCC 也是会被匹配到的，但是，题目要求的小写字母是前后严格只有三个大写字母，所以还需要补上 [^A-Z] ，这表示非大写字母

总的模式就是：[^A-Z]+[A-Z]{3}([a-z])[A-Z]{3}[^A-Z]+，这个 + 写不写无所谓，就本题而言，主要是 aAAAbCCCc 的都行，无所谓前后是大写还是小写。

结果： 

linkedlist

修改 url： 

修改前：http://www.pythonchallenge.com/pc/def/equality.html
修改后：http://www.pythonchallenge.com/pc/def/linkedlist.html

访问修改后的 url 得到提示 linkedlist.php ，访问http://www.pythonchallenge.com/pc/def/linkedlist.php 即可。

页面源码：

点击图片发现可以跳转：

注意 url 中的 nothing，应该是通过不停的获得 nothing 然后跳转到下一个页面，最后某个页面会出来答案。手动几次后发现确实如此：

题目信息

页面源码：

注意绿字提醒：python 中有一个库叫做 pickle，用来序列化和非序列化的。因此，本题需要使用到该库。还有 banner.p 点进去发现是看不懂的一堆字符串，做一个合理的猜测：banner.p 中的字符串是经过序列化的，将它反序列应该会得到答案

方法

代码： 

import requests, pickle  
url = 'http://www.pythonchallenge.com/pc/def/banner.p'  
raw = requests.get(url).content  
print(pickle.loads(raw))

发现得到看不懂的一堆东西：

仔细观察发现，是一个二元列表，每一行都是由一个或多个元组组成的，猜测（看完答案后）：每一行其实都是字符串的压缩：将一行字符串压缩成 [(字符1，出现次数), (字符2，出现次数)...] 因此，只要将每一行还原成原来的样子即可。

最终代码： 

import requests, pickle  
url = 'http://www.pythonchallenge.com/pc/def/banner.p'  
raw = requests.get(url).content  # 转为 byte，因为 loads() 的参数是 byte 类型的
data = pickle.loads(raw)  
for line in data:  
    print("".join([ch * num for ch, num in line]))

结果： 

channel

level 6

题目信息

查看页面源码：

注意：html 旁边的 <!-- <-- zip -->。尝试将 html 修改为 zip 发现可以下载一个压缩文件。

压缩文包里有许多文件，先看 readme.txt： 

welcome to my zipped list.  
  
hint1: start from 90052  
hint2: answer is inside the zip

查看之后猜测，除 readme.txt 之外每一个文件内容都是： 

Next nothing is (\d+)

思路：nothing 从 90052 开始，先访问对应文件，然后读取下一个 nothing 继续访问。直到出现不同

方法

第一次代码： 

import re  
rpath = "attachments/level_6/%s.txt"  
nothing = "90052"  
pattern = "Next nothing is (\d+)"  
while True:  
    path = rpath % nothing  
    with open(path, 'r') as f:  
        content = f.read()  
        print(content)  
        match = re.search(pattern, content)  
        if not match:  
            break  
	nothing = match.group(1)

解释： match.group(1)：因为代码中的 pattern 里面只有一个括号（在我们需要处加上括号），这样子匹配下来的其实是一整句话 

Next nothing is (\d+)

但是，我们只需要 () ，如果直接使用 match.group() 就会将整句话给 nothing，而 group(1) 就是只读取第一个括号中的内容，也就是 () 即 nothing。

不出所料，果然没这么简单，当 nothing = 46145 时：

现在的问题是什么是 comments？经过百度后发现 zip 文件有 comment，用来写额外的描述数据。python 中有 zipfile 库可以获得 comment。

最终代码： 

import re, zipfile  
path = "attachments/level_6/channel.zip"  
nothing = "90052"  
comment = []  
pattern = "Next nothing is (\d+)"  
with zipfile.ZipFile(path, "r") as f:  
    while True:  
        fileName = nothing + '.txt'
        content = f.read(fileName).decode("utf-8")  
        comment.append(f.getinfo(fileName).comment.decode('utf-8'))  
        match = re.search(pattern, content)
        if not match:  
            break
        nothing = match.group(1)  
    print("".join(comment))

结果：

访问 http://www.pythonchallenge.com/pc/def/hockey.html，得到： 

it's in the air. look at the letters.

注意到，组成 HOCKY 的字母为 oxygen，因此修改 url： 

修改前：http://www.pythonchallenge.com/pc/def/hockey.html
修改后：http://www.pythonchallenge.com/pc/def/oxygen.html

随便取一行即可，这里取 47，因为 img.height / 2 的值为 47。

第三步，取出 47 行的所有像素点： 

from PIL import Image
path = "attachments/level_7/oxygen.png"

发现，除了第一个之外，其余的像素点都重复了 7 次，因此我们剔除重复的：（上面的代码稍作修改） 

from PIL import Image
path = "attachments/level_7/oxygen.png"
img = Image.open(path)
row = [img.getpixel((x, img.height/2)) for x in range(img.width)]
for i in range(0, img.width, 7): # 修改了这里
    print(row[i])

第四步，每个灰度图像都有一个灰度值，因此做一个猜测：将这些灰度值当作 ascii 码，然后转为字符。 

from PIL import Image
path = "attachments/level_7/oxygen.png"
img = Image.open(path)  
row = [img.getpixel((x, img.height/2)) for x in range(img.width)]

for i in range(0, img.width, 7):

print("".join(info))

结果： 

smart guy, you made it. the next level is [105, 110, 116, 101, 103, 114, 105, 116, 121]pe_

后边的 pe_ 我猜测是因为第一个像素点就重复了5次，而我取步长为7，导致每次取得都不是重复值里面得第一个，因为可能将重复值后面得无关的也取了进来。

将上面结果中得数字也提取出来，当作 ascii 码处理，因此最终代码如下： 

import requests, re  
from PIL import Image  
img_url = "http://www.pythonchallenge.com/pc/def/oxygen.png"  
path = "attachments/level_7/oxygen.png"  
img = Image.open(path)  
row = [img.getpixel((x, img.height/2)) for x in range(img.width)]  
info = []  
for i in range(0, img.width, 7):  
    info.append(chr(row[i][0]))  
message = "".join(info)  
next = re.findall(r'(\d+)', message)  
for c in next:  
    v = int(c)  
    print(chr(v), end='')

结果： 

integrity

level 8

题目信息

查看页面源码：

发现：

  1. 图片中的蜜蜂可以点开，点开之后会跳出一个验证界面
  2. 绿色字符串正好对应的用户名和密码

但是，很明显可以看出应该是经过某种编码的，后来发现，是通过 bz2 压缩得到的。而 python 自带了 bz2 库。

方法

代码如下： 

import bz2  
un = b'BZh91AY&SYA\xaf\x82\r\x00\x00\x01\x01\x80\x02\xc0\x02\x00 \x00!\x9ah3M\x07<]\xc9\x14\xe1BA\x06\xbe\x084'

level 10

题目信息

注意到 len(a[30]) = ?。查看页面源码，发现图片中的牛可以点击，并且得到一个数组 a： 

a = [1, 11, 21, 1211, 111221,

通过观察，发现这个数组并没有闭合。那么思路就是：通过数组的某种规律得到 a[30] 的值，然后再计算长度。

通过查资料发现，这是外观序列。

外观序列（Look-and-say sequence），是指以下特点的整数序列：

1, 11, 21, 1211, 111221, 312211, 13112221, 1113213211, ……

它以数字1开始，序列的第n+1项是对第n项的描述。比如第2项是2个1，所以下一项（第三项）就是21。又比如第5项是111221，描述就是3个1，2个2，1个1， 所以下一项就是312211。

方法 1

代码如下： 

def getNext(res):  
    index, next = 0, ""  
 while index < len(res):  
	count = 1  
	while index < len(res) - 1 and res[index] == res[index+1]:  
		count += 1  
		index += 1  
	next += str(count) + res[index]  
	index += 1
return next

def countAndSay(length):  
    res = '1'  
	for i in range(length-1):  
		res = getNext(res)
	return(res) 

print(len(countAndSay(31)))

（注意，一定是 count 在前，res[index] 在后）

得到结果：5808

因此修改 url： 

修改前：http://www.pythonchallenge.com/pc/return/bull.html
修改后：http://www.pythonchallenge.com/pc/return/5808.html

方法 2

利用正则表达式替代循环。 

import re
print(re.findall(r'(\d)(\1*)', '111221'))

结果为： 

[('1', '11'), ('2', '2'), ('1', '')]

（假设元组第一个元素为 i，第二个元素为 j）那么与方法 1相比： 

value == res[index]
count == len(str(i+j))

因此，最终代码如下： 

import re  
res = '1'  
for i in range(30):  
    res = "".join([str(len(i+j))+i for i, j in re.findall(r'(\d)(\1*)', res)])  

print(len(res))

out = Image.new('RGB', (100, 100))  
d = 200  
delta = [(1, 0), (0, 1), (-1, 0), (0, -1)]  
x, y, i = -1, 0, 0  # 因为第一次要 +100，所以 x 初始为 -1
while d/2 > 0:  
    for v in delta:  
        steps = d // 2
		for s in range(steps):  
            x, y = x + v[0], y + v[1]  
            out.putpixel((x, y), img.getpixel((i, 0)))  
            i += 1
		d -= 1  
out.show()
