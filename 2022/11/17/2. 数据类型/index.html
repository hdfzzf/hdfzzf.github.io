1. 简介

PHP 支持 8 种原始的数据类型，其中分为：

  1. 4 种标量类型；
    1. Boolean
    2. Interger
    3. Float 或者 double
    4. String
  2. 2 种复合类型；
    1. Array
    2. Object
  3. 2 种特殊类型；
    1. Resource
    2. NULL

为了确保代码的易读性，还有一些伪类型：

  1. Mixed
  2. Number
  3. Callback

关于变量的值与类型的操作：

  1. 查看某个变量的 类型，可以使用函数 gettype(变量名)；
  2. 查看某个变量的 值和类型，可以使用函数 var_dump(变量名)；
  3. 强制改变类型，但原变量的类型没有发生改变，而是生成新的指定类型的变量，方式 $new_var = (type)var；
  4. 永久性改变类型，直接改变原变量的类型，而不生成新的变量，使用函数 settype(var, type)；
  5. 判断变量的类型，可以使用 is_type(var)；

  3. null
  4. 空列表；
  5. 单字符 0；

3. Integer

Integer 不只包含十进制，还可以表示 8 进制和 16 进制。其中：

  1. 8 进制以 0 开头；
  2. 16 进制以 0x 开头；

根据结果也可以知道，不论是什么进制的数，最终都是以十进制方式显示。

5. String

字符串可以用 4 中方式表达：

  1. 单引号；（单引号中有单引号需要转义 \）
  2. 双引号；（双引号中有双引号也需要转义 \）
  3. heredoc 语法结构；
  4. nowdoc 语法结构；（自 PHP 5.3.0 起）；

介绍第一种，单引号创建的字符串就是普通的字符串，如果字符串中出现单引号，需要使用 \ 转义。

使用双引号创建的字符串与单引号创建的最大不同：可以解析变量。但遇到双引号也需要进行转义。

可以看到 $name 被成功解析。

变量也可以不用 {} 包含，但是为了与字符作区分，为了易读性，所以用 {}。

使用 heredoc 语法创建的字符串与用双引号创建的相同，也可以解析变量，不同就是出现双引号不用转义。

需要注意的是 写法：

  1. <<<LIST 怎么写无所谓，可以直接写在等号后面，也可以另起一行，还可以 <<< LIST；
  2. 结尾的 LIST 需要注意，最好顶格写，如果不顶格写，需要保证 LIST 之间的内容的缩进要比结尾的 LIST 多；

使用 nowdoc 创建的字符串与单引号相同，无法解析变量，也可以与双引号相同，解析变量，这取决于开头的 <<<'EOD' 是单引号还是 <<<"EOD" 双引号。与 heredoc 类似，遇到单引号也不需要转义。

也需要注意 写法：

  1. EOD 和 <<<'EOD' 与 heredoc 一样的规则；
  2. 除此之外，结尾的 EOD 不需要加引号；
  3. 更奇妙的是，如果开头的 EOD 使用了双引号，则可以起到与 heredoc 一样的效果；

6.2. 数组的分类

根据键名的类型，主要分为两类：

  1. 键名全是整数：索引数组；（数组）
  2. 键名全是字符串：关联数组；（字典）

如果键名不是这两类，则会强制转换。但 PHP 不区分索引数组和关联数组，也就是说，一个数组中的键可以同时存在整数和字符串，但如果出现其他的，则会强制转换。

解释：

  1. 1 => 10 创建了键值对 1 => 10；
  2. 1.4 => 20，因为键时浮点数，转为整数（舍弃小数），因此 1 => 20，所以此时更新键值对 1 => 20；
  3. true => 30，同 1.4 => 20，true 转换为整数 1，因此更新键值对 1 => 30；
  4. 最后 0.3 => 40，浮点数转为整数 0，创建键值对 0 => 30；
